Vue系列-知识总结

本篇是一个Vue体系知识的回顾篇，主要是想对Vue的体系知识有一个大概的总结，在自己脑海里梳理成一个体系。

Vue生命周期

官方生命周期图示

可以看到，在Vue实例的生命周期的各个阶段中，都会有相应的钩子函数提供给开发者来操作。我们来一一解读这些钩子。

  • beforeCreate。钩子触发的时机是实例创建之后，但是数据和事件还未配置之前调用；
  • created。如果实例创建完成，并且data，prop，方法和watch等已经配置完成，就会触发该钩子；
  • beforeMount。在挂载之前触发，相关的render函数会首次被调用；
  • mounted。新创建的虚拟DOM替换掉真实DOM，并挂载在实例上之后，就会触发该钩子；
  • beforeUpdate。当数据发生变化，在虚拟DOM重新渲染之前，会触发该钩子；
  • updated。由于数据发生变化，并且虚拟DOM重新渲染完毕，就会触发该钩子，这时DOM已经更新完毕；
  • activated。当keep-alive组件激活时调用，如果是网络请求，可以放在此处；
  • deactivated。当keep-alive组件停用时会调用，如果需要取消网络请求，可以放在此处；
  • beforeDestroy。Vue实例被销毁之前调用，但是该钩子内部，实例还是存在的，可以做一些业务销毁动作；
  • destroyed。Vue实例销毁之后调用，此时所有的子实例也会被销毁。

子组件的生命周期

如果是单个Vue实例的生命周期，如同上图那样，但是，如果存在子组件，那么其执行顺序到底是什么样的呢，我们来一探究竟。

  • 渲染过程

    父组件(beforeCreate) -> 父组件(created) -> 父组件(beforeMount) -> 
子组件(beforeCreate) -> 子组件(created) -> 子组件(beforeMount) ->
子组件(mounted) -> 父组件(mounted)

  • 更新过程

    父组件(beforeUpdate) -> 子组件(beforeUpdate) -> 子组件(updated) -> 父组件(updated)

// 如果只是父组件更新
父组件(beforeUpdate) -> 父组件(updated)

  • 销毁过程

    父组件(beforeDestroy) -> 子组件(beforeDestroy) -> 子组件(destroyed) -> 父组件(destroyed)
