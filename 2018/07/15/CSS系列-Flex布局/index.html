CSS系列-Flex布局

时至今日，能够稳定的实现跨浏览器的布局方式还是float和position，但是对于一些特殊布局，实现上十分不方便。所以目前来讲，Flex布局应用也是越来越广泛，这篇文章的主题就是一步步学会Flex布局的使用。

指定元素布局为Flex

首先，如果想要某个元素使用flex布局，首先需要指定该元素的display值：

.content {
    display: flex;
}

如果是行内元素，可以设置为：

.content {
    display: inline-flex;
}

Flex模型

我们以MDN上的一张图来作为参考：

这张图说明了FLex布局的一些关键点：

  • main axis。是布局主轴，是Flex元素延伸的轴，该轴的开始和结束称为main start和main end；
  • cross axis。是垂直于flex元素延伸的轴，该轴的开始和结束称为cross start和cross end；
  • 如果指定display值为flex，那么其容器就称为flex container；
  • 在flex container中每个子flex元素称为flex item。

Flex-Direction

对于flex item的排列方式，有两种：行和列。默认是以行的方式去横向排列，如同上面的图示，但是flex也提供了列的方式去实现。

flex-direction: row;    // 横向往右方向去排列
flex-direction: row-reverse;    // 横向往左方向去排列
flex-direction: column;    // 竖向往下方向去排列
flex-direction: column;    // 竖向往上方向去排列

Flex-Wrap

随着flex item的变多，如果布局使用了定宽和定高，那么就会发生变形，带来的效果就是高宽可能会被压缩。所以，这里需要做的变动就是当高或宽受到限制的时候，为了flex item的高宽不发生变化，及时去做换行处理。

flex-wrap: wrap;

这样做确实换行了，但是可能最后一行的flex item并没有铺满一行，效果有点突兀。所以我们需要指定flex item的flex值，设定个最小值。

flex: 100px;

这里的flex-wrap是换行策略，有三个值可以设定。

flex-wrap: wrap;
flex-wrap: nowrap;    // 默认
flex-wrap: wrap-reverse;

Flex-Flow

上面我们定义了flex-direction和flex-wrap两个属性，如果使用flex-flow的话，可以把它们放在一起使用。

flex-flow: row wrap;

也就是说，flex-flow是flex-direction和flex-wrap的缩写。

Flex

如果需要指定flex item的尺寸的时候，可以使用width和height来做，但是对于动态尺寸，我们可以用flex属性来实现。

比如，我们有4个flex item，前三个flex值都是1，最后一个是3，那么前三个就会平分一半空间，最后一个独占一半空间。

.item {
    flex: 1;
}
.item:nth-of-type(4) {
    flex: 3;
}

还有另外一种写法就是下面这样：

.item {
    flex: 1 400px;
}

它表示的是会首先给每个item分配400px的空间，然后将剩余的可用空间在按照分配的比例给共享。

这样做好处很多，因为没有设定宽高就实现了动态尺寸，所以如果浏览器的尺寸发生变化，或者是动态生成flex item，都不会对原有布局造成影响。

当然，flex属性其实是flex-grow、flex-shrink和flex-basis的集合。

Align-Items

align-items是控制交叉轴的对齐方式。一共有5种效果。

align-items: flex-start;    // 交叉轴起点对齐
align-items: flex-end;        // 交叉轴终点对齐
align-items: center;        // 交叉轴中心对齐
align-items: baseline;        // 以项目的第一行文字的基线对齐
align-items: stretch;        // 如果项目未设置高度或者为auto，那么将铺满整个容器

Justify-Content

和align-item类似，justify-content控制的是主轴的对齐方式，也是有5种效果。

justify-content: flex-start;    // 以主轴起点处对齐
justify-content: flex-end;        // 以主轴结束点对齐
justify-content: center;        // 以主轴中心对齐
justify-content: space-between;    // 两端对齐，项目间隔等分
justify-content: space-around;    // 每个项目的间隔等分

Align-Content

如果flex容器中存在多根轴线，那么align-content的效果就和多个align-items的集合一样，效果可以自行测试一下。

Order

如果需要将flex item做下排序，而不影响到dom里的节点顺序，那么可以使用这个属性去实现。

order: 0;    // 默认

order值大的会显示靠后点。

Align-Self

之前的align设定都是基于flex容器来的，那么如果某个flex item想单独地使用某个align方式，可以使用这个属性完成，效果可以参考align-items来。

总结

基本的Flex布局的介绍就这么多了，相比盒子模型，还算好理解点，目前，浏览器上的支持也是越来越完善，是时候使用flex布局来完成绝大部分需求了。
